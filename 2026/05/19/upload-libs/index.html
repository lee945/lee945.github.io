Pass 10：黑名单绕过（点空格点，deldot 逻辑绕过）

源码

分析

三重防护（trim + deldot + str_ireplace ::$DATA）同时存在，但 deldot() 有逻辑缺陷：

  • deldot() 从后向前检测，遇到空格会停止扫描，导致空格后的点未被删除
  • 构造 shell.php. . → deldot 只删最后一个点 → 得到 shell.php. （末尾为空格+点）
  • strrchr 取最后一个 . 之后 → 得到 " ."
  • trim() → 得到 "."，不在黑名单中
  • Windows 保存时自动清理末尾的 . → 只剩 shell.php

Payload

抓包，将文件名改为 shell.php. .（点 + 空格 + 点）：

连接成功：

要点： deldot() 遇到空格停止的设计缺陷导致可构造 文件名. . 绕过。正确的去点逻辑应循环删除直至末尾没有点，不受空格影响。

连接成功：

要点： str_ireplace 一次性替换，不递归。双写可绕过。防御侧应直接将替换改为拦截（返回错误），而非尝试”清洗”文件名。

源码

分析

  • 校验方式： 后缀白名单（jpg/png/gif），且后缀取自 strrpos + substr，无法通过文件名绕过
  • 新攻击面： save_path 由 GET 参数控制，可注入 %00 截断符，截断后续的随机数 + 后缀拼接
  • 利用条件： PHP < 5.3.4，magic_quotes_gpc = Off

Payload

上传 shell.php → 抓包 → 文件名改为 shell.png（过白名单）→ save_path 改为 ../upload/shell.php%00：

要点： %00 是 C 字符串结束符，PHP 底层 move_uploaded_file 使用 C 函数处理路径，遇到 \0 即截断，后面拼接的随机数 + .png 被丢弃。防御端应从 $_GET['save_path'] 的传参转移到代码内部硬编码。

源码

分析

  • 校验方式： 读取文件前 2 字节，比对文件幻数（Magic Number）
  • 防御强度： 比黑名单后缀高，直接检查文件真实类型

  1. 图片马中的 PHP 代码被执行，phpinfo() 输出可见：

要点： 文件头校验能防直接上传脚本，但结合文件包含（include/require）即可绕过。终极防御需同时做到：① 文件头校验 ② 重命名文件名（随机散列）③ 上传目录关闭脚本执行 ④ 限制文件包含路径。

Pass 15：同 Pass 14

同 Pass 14，同样是文件头校验 + 文件包含的组合，绕过方法不变（图片马 + include.php）。

要点： 与 Pass 14 相同，图片马 + 文件包含仍是可行绕过路径。

总结

Pass 校验方式 过滤规则 绕过方法
1 前端 JS 校验 白名单 .jpg/.png/.gif 禁用 JS
2 MIME-Type 校验 image/jpeg, image/png, image/gif 抓包改 Content-Type
3 黑名单（不完整） .asp/.aspx/.php/.jsp 换 .php3 后缀
4 黑名单（缺 .htaccess） 50+ 后缀，含 .ini 上传 .htaccess + AddType
5 黑名单（缺 .user.ini） Pass4 + .htaccess .user.ini + auto_prepend_file
6 黑名单（缺 strtolower） 含大小写变体，但未统一转小写 大写 .Php
7 黑名单（缺 trim） 三重防护但缺 trim 末尾加空格
8 黑名单（缺 deldot） 三重防护但缺 deldot 末尾加点
9 黑名单（缺 ::$DATA） 三重防护但缺 ::$DATA 过滤 追加 ::$DATA
10 黑名单（deldot 逻辑缺陷） 三重防护全有 .php. .（点空格点）
11 str_ireplace 替换为空 替换黑名单字符串为空 双写 pphphp
12 白名单 + GET 截断 仅 jpg/png/gif save_path + %00 截断
13 白名单 + POST 截断 同 Pass 12，POST 传参 Hex 00 截断
14 文件头幻数校验 前 2 字节 Magic Number 图片马 + 文件包含
15 同 Pass 14 同 Pass 14 图片马 + 文件包含

核心思路总结：

  1. 客户端校验不可信 — 禁用 JS 或抓包改包即可绕过
  2. MIME-Type 来自请求头 — 由客户端控制，不可作为防御依据
  3. 黑名单永远不完整 — 特殊后缀、配置文件（.htaccess/.user.ini）、大小写、空格、点、流标识符，总能找到遗漏
  4. 替换清理有逻辑缺陷 — deldot 遇空格停止、str_ireplace 只替换一次
  5. 路径拼接不可控 — %00 / Hex 00 截断符可操纵文件保存路径
  6. 文件头校验可结合文件包含绕过 — 图片马 + include 代码执行
  7. 终极防御方案： 白名单后缀 + 文件头校验 + 随机文件名 + 上传目录关闭脚本执行 + 限制文件包含
