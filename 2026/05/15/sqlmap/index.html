SQLMap 使用指南

开源自动化 SQL 注入工具

SQLMap 概述

SQLMap 是一款开源的自动化 SQL 注入检测与利用工具，支持多种数据库类型：

MySQL、Oracle、Microsoft Access、PostgreSQL、SQL Server、IBM DB2、SQLite、Firebird、Sybase、SAP MaxDB

SQLMap 支持以下 5 种 SQL 注入技术：

缩写 注入技术
B Boolean-based blind SQL injection（布尔盲注）
E Error-based SQL injection（报错注入）
U UNION query SQL injection（联合查询注入）
S Stacked queries SQL injection（堆叠查询注入）
T Time-based blind SQL injection（时间盲注）

Level 与 Risk 详解

Level（检测等级：1-5，默认 1）

Level 决定了 sqlmap 检测的深度和范围：

Level 检测范围
1 仅 GET/POST 参数
2 GET/POST + HTTP Cookie
3 GET/POST + Cookie + HTTP User-Agent / Referer
4 以上全部 + 更多 payload 变体
5 以上全部 + 最全面的 payload 覆盖（含 Host 等）

Level 越高，检测越全面，但耗时会显著增加。

Risk（风险等级：1-3，默认 1）

Risk 决定了 payload 的危险程度：

Risk 说明
1 安全，无明显风险
2 使用部分可能造成数据影响的 payload
3 使用 OR 语句、UPDATE 等高风险 payload

Risk 3 的 OR 语句可能导致全表更新，生产环境慎用。

参数 作用
-u 指定目标 URL
-r 从文件加载 HTTP 请求

-g Google 搜索结果扫描
-p 指定注入参数
--dbs 枚举所有数据库
--current-db 获取当前数据库名
-D 指定数据库
-T 指定表
-C 指定字段
--columns 枚举字段
--tables 枚举表
--dump 导出数据
--dump-all 导出所有数据
--users 枚举数据库用户
--passwords 枚举用户密码 hash
--current-user 获取当前用户
--is-dba 判断是否为 DBA
--roles 获取用户角色
--batch 使用默认选项，无需交互确认
--dbms=MySQL 指定数据库类型
--technique=B 指定注入技术（B/E/U/S/T）
--level=3 设置检测等级（1-5）
--risk=3 设置风险等级（1-3）
--os-shell 获取操作系统 shell
--sql-shell 获取 SQL 交互 shell
--cookie 指定 Cookie
--data POST 方式提交数据
--param-del 参数分隔符（默认 &）
--tamper 使用绕过脚本
--random-agent 随机 User-Agent
--proxy 使用代理（如 --proxy="127.0.0.1:8080"）
--threads 5 多线程（默认 1）
--fresh-queries 不使用缓存，每次重新查询
--flush-session 清空 session 缓存
--keep-alive HTTP 持久连接
--hex 使用十六进制传输数据
--no-cast 关闭强制转换机制
--mobile 使用移动端 User-Agent
-v3 详细输出模式（显示 payload）
-h 查看帮助（-hh 查看完整帮助）
--update 更新 sqlmap（需要 git）
