Less-8：布尔盲注

背景

Less-8 页面存在 SQL 注入，但没有数据显示位，也没有数据库报错回显。页面只在查询结果为空时显示 You are in...........，有结果时无任何输出——即只能通过页面返回”正常/异常”两种状态进行判断。

此时需要布尔盲注：逐字符比较 ASCII 值，根据页面响应判断 True/False，逐步猜解出完整数据。

要点： 布尔盲注的核心是利用页面不同状态（正常/异常）作为二进制的 True/False 信号，逐一爆破每个字符。配合 Burp Suite Intruder 的 Cluster Bomb 模式可同时对字符位置和 ASCII 值进行爆破，大幅提升效率。

Less-9：时间盲注

背景

Less-9 比 Less-8 更进一步：页面始终返回相同内容，无论查询结果为空还是非空，页面都没有任何差异。此时连布尔判断都无法进行，只能利用时间延迟来推断条件是否成立。

核心思路：将条件包裹在 if(条件, sleep(5), 1) 中——条件为真则延迟 5 秒，条件为假则立即返回。通过响应时间的长短判断条件是 True 还是 False。

1. 猜解数据库长度

页面延迟 5 秒 → 数据库长度为 8。

2. 猜解数据库名

延迟 5 秒 → 第一位是 s（ASCII 115）。

逐字符爆破得到数据库名：security。

3. 猜解表的数量

延迟 5 秒 → security 库共有 4 张表。

4. 猜解表名

延迟 5 秒 → 第一个表名的首字母是 d（ASCII 100），逐字符爆破得到第一个表名为：emails。

后续猜解剩余表名、列名、获取数据的过程与布尔盲注完全一致，只需将布尔条件整体放入 if(条件, sleep(5), 1) 中即可。

5. 后续爆破（示例）

与布尔盲注的关系： 时间盲注本质上是布尔盲注的延伸——只需将布尔条件包裹在 if(条件, sleep(5), 1) 中。条件为真 → 延迟 N 秒；条件为假 → 快速返回。根据响应时间即可逐一爆破所有数据。时间盲注的缺点是速度慢（每个字符至少等待 sleep 时长），适合在无任何回显差异时使用。
