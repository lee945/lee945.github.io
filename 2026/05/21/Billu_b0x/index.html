六、权限提升

6.1 漏洞原理

CVE-2015-1328 是 Linux kernel overlayfs 文件系统的本地提权漏洞。ovl_setattr 函数在设置文件属性时未正确检查权限，允许非特权用户创建具有 root 所有权的文件，进而通过 SUID 二进制文件提权。

七、总结

攻击链回顾

关键漏洞点

漏洞 说明 危害
敏感信息泄露 /in.php 直接输出 phpinfo() 泄露服务器配置、绝对路径
文件包含 (LFI) test.php 未限制包含路径 读取任意 PHP 源码及系统文件
源码泄露 通过 LFI 读取 index.php / add.php 逻辑 暴露 SQL 查询和上传处理逻辑
SQL 注入 单引号过滤可被 \ 转义绕过 登录绕过进入后台
文件上传过滤不足 仅检查 MIME + 扩展名 + 文件头 图片马绕过，上传 PHP webshell
文件包含执行 panel.php ?load= 参数未限制 可包含上传的图片马实现 RCE
内核提权 CVE-2015-1328 (overlayfs) www-data → root

反思

  1. LFI 是信息收集的最高效手段 — 一个 test.php 的文件包含漏洞直接读出了登录逻辑和上传逻辑的源码，相当于把整个应用的设计摊开给你看
  2. 源码审计比盲目 fuzz 高效 — 读 index.php 源码后，\ 绕过单引号过滤的思路变得显而易见，比用 sqlmap 跑字典更有针对性
  3. 图片马几乎防不住 — 只要允许上传图片且上传目录能执行 PHP，MIME + 扩展名 + 文件头三层过滤等于形同虚设
  4. 文件包含 + 文件上传组合拳 — 单独的文件上传或文件包含可能都不致命，但两者结合是最快 getshell 路径
  5. 老系统提权靠内核 — Ubuntu 12.04 + kernel 3.13 的公开 exploit 命中率极高，searchsploit 一把出结果
  6. 权限目录要记住 — /var/www 无写入权限时，/tmp 是恒久的下载/编译阵地
