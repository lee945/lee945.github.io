xss-labs 靶场笔记

cHr1s2026-05-182026-05-18

逐关记录 xss-labs 靶场 Level 1 到 Level 16 的通关过程，包括每关源码分析、过滤规则、Payload 构造思路和截图验证。

涉及关卡：Level 1（基础反射型 XSS）、Level 2（属性注入 + 双引号闭合）、Level 3（单引号闭合 + 事件触发）、Level 4（尖括号过滤 + 事件触发）、Level 5（<script>/on 关键字过滤 + <a> 标签 href 注入）、Level 6（href/src/data 关键字过滤 + 大小写绕过）、Level 7（关键字空替换 + 双写绕过）、Level 8（HTML 实体编码绕过）、Level 9（http:// 检测 + 注释符绕过）、Level 10（hidden 参数 + 事件触发）、Level 11（HTTP Referer 头注入）、Level 12（HTTP User-Agent 头注入）、Level 13（Cookie 注入）、Level 14（exifviewer 已失效）、Level 15（AngularJS ng-include 注入）、Level 16（空格/script/斜杠/制表符替换为 &nbsp; + %0A 换行符绕过）。Level 17-20 需要 Flash 插件，已废弃不做。

Level 9：http:// 检测 — HTML 实体编码 + 注释符绕过

源码

分析

  • 在 Level 8 基础上新增：必须包含 http:// 字符串，否则拒绝输出链接

  • 仍然使用 HTML 实体编码绕过关键字过滤

Payload

技巧： 利用 JS 注释 /* */ 消化掉必须存在的字符串，既满足后端检测又保持 payload 有效。

源码

分析

需要先将 hidden 改为 text 使其可见，然后鼠标滑过输入框触发。

Level 15：AngularJS ng-include 注入

源码

分析

  • 使用了 AngularJS（<html ng-app>）
  • ng-include 指令会加载并包含外部 HTML 片段
  • 参数 src 的值经过 htmlspecialchars() 后拼入 ng-include 属性
  • ng-include 可以包含同域下的任意 PHP 页面，利用 Level 1 的无过滤 XSS 作为跳板

Payload

通过 ng-include 加载 Level 1 页面，并在 name 参数中注入 XSS payload。

Level 16：空格/script/斜杠过滤 — %0A 换行符绕过

源码

分析

  • 输出位置：<center> 标签内，直接拼接，无 htmlspecialchars 转义
  • 过滤规则（替换为 &nbsp;）：
  • script 关键字和空格均被破坏，<script> 标签无法使用
  • 空格和 / 被替换为 &nbsp;，<img src=1 onerror=...> 这类标签也无法使用
  • 利用 %0A（换行符） 替代空格分离标签和属性
  • <svg> 标签支持 onload 事件，且无需 /> 闭合

Payload

  • %0A（LF 换行符）未被过滤，可替代空格
  • <svg onload=alert(1)> 不依赖 src 属性和闭合斜杠

技巧： 空格被过滤时，%0A（LF）、%0D（CR）、%0C（换页）都可作为空格替代。<svg onload=alert(1)> 不依赖 src 和闭合斜杠，比 <img> 适应性更强。

Level 17-20：Flash 插件（已废弃）

Level 17 到 Level 20 基于 Adobe Flash 插件实现 XSS，Flash 已于 2020 年底停止支持，现代浏览器不再支持，故跳过不写。

总结

Level 注入类型 传参位置 过滤规则 绕过技巧
1 基础反射型 XSS GET name 无 直接注入 <script>
2 属性注入（双引号闭合） GET keyword htmlspecialchars() 仅转义输出位 闭合 " 注入 <script>
3 事件触发绕过 htmlspecialchars GET keyword htmlspecialchars() 不转义单引号 + <script> 过滤 ' 闭合 + onmouseover/onclick
4 尖括号过滤绕过 GET keyword > < 替换为空 " 闭合 + onmouseover
5 关键字过滤（加下划线） GET keyword <script/on 被破坏 + 强转小写 <a href="javascript:...">
6 扩展关键字过滤 GET keyword <script/on/src/data/href 被破坏 大小写混合绕过
7 关键字空替换 GET keyword script/on/src/data/href 删除 + 强转小写 双写绕过（sscriptcript）
8 href 属性注入 GET keyword 关键字加下划线 + 双引号转义 + 强转小写 HTML 实体编码绕过
9 http:// 检测绕过 GET keyword Level 8 规则 + 必须含 http:// HTML 实体编码 + /* http:// */ 注释
10 隐藏参数注入 GET t_sort > < 删除 " 闭合 + onmouseover
11 Referer 头注入 HTTP Referer > < 删除 闭合 " + onfocus
12 User-Agent 头注入 HTTP User-Agent > < 删除 闭合 " + onfocus
13 Cookie 注入 Cookie user > < 删除 闭合 " + onfocus
14 第三方平台利用 iframe 外部网站 — 网站已失效
15 AngularJS ng-include 注入 GET src htmlspecialchars() ng-include 跨页面加载 XSS
16 空格/script/斜杠过滤 GET keyword script/空格///制表符 → &nbsp; + 强转小写 %0A 换行符替代空格 + <svg onload>
17-20 Flash XSS（已废弃） — — Flash 已停止支持，不再可用

核心思路总结：

  1. 先看输出位置：在标签内容中、属性值中、还是 JS 代码块中？这决定了闭合方式。
  2. 再看过滤规则：哪些字符/关键字被处理了？是删除、替换还是转义？
  3. 选择绕过策略：
    • 标签/事件被过滤 → 换标签（<a>、<img>、<svg>）
    • 大小写敏感过滤 → 大小写混合
    • 空替换只执行一次 → 双写绕过
    • 属性值编码解码差异 → HTML 实体编码
    • 空格被过滤 → %0A / %0D / %0C 换行换页符替代
    • GET 参数被转义 → 尝试 HTTP 头（Referer/UA/Cookie）
